# HomeMade - Product Guidelines

## Core Principles
- **Clarity Over Complexity:** Every UI element must serve a clear purpose. Avoid clutter to ensure users can manage their pantry quickly.
- **Mobile-First Utility:** Designed primarily for use in the kitchen, meaning high-contrast elements and touch-friendly targets are essential.
- **Resilient AI Interaction:** AI suggestions should feel like a helpful assistant. Provide clear feedback during generation, handle fallback mechanisms, and offer process transparency (model name, generation status) to build user trust.

## Visual Design
- **Typography:** Use **Geist** for a modern, tech-forward feel. Support high readability with appropriate line-heights and contrast.
- **Color Palette:** A clean, neutral base (whites/greys) with meaningful accent colors for categories (e.g., green for vegetables, red for meats).
- **Layout:** Strictly **RTL (Right-to-Left)** for the Hebrew interface. Ensure consistent spacing and alignment across all screen sizes.

## User Experience (UX)
- **Zero-Waste Workflow:** Minimize the number of taps required to add or consume an ingredient.
- **Visual Cues:** Use icons and color-coding for ingredient categories to allow for rapid visual scanning.
- **Real-Time Sync:** Ensure changes are reflected immediately across all devices to prevent data conflicts in shared households.

## Language & Tone
- **Professional yet Friendly:** The tone in Hebrew should be approachable but precise, using common culinary terms rather than overly technical jargon.
- **Action-Oriented:** Use clear, imperative labels for buttons (e.g., "Add", "Generate", "Cook").
